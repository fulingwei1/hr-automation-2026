# 2026年1月 - 人力行政部自动化优化行动清单

> 时间：2026年1月 | 目标：在1月完成基础准备，为2月-3月的优化奠定基础

### 【HR团队】数据标准化

- [ ] **1.4** 盘点招聘渠道清单 ⏰ 1月7日

  - 责任：HR人员

  - 交付：反馈清单（3-5条改进建议）
  - 责任：符凌维or 舒盼收集整理

## 第二周（1月13-17日）：快速赢 - 候选人追踪系统

### 【重点】建立候选人信息统一汇总

- [ ] **2.1** 整理现有的招聘情况 ⏰ 1月13日
  - 工作量：2小时
  - 内容：盘点当前正在进行的所有招聘岗位（岗位名、发布时间、简历数等）
  - 交付：当前招聘岗位清单
  - 责任：招聘主管

- [ ] **2.2** 创建"招聘跟踪汇总表" ⏰ 1月13-14日
  - 工作量：3小时
  - 内容：
    - 所有简历统一汇总到一张表
    - 标记每个候选人的当前状态（已通知/待反馈/已面试/已背调/已Offer等）
    - 记录最后更新时间和负责HR
  - 交付：Excel表格 + 使用规范
  - 责任：HR团队协作完成
  - 工具：使用现有的resume-screener进行评分，在表里补充追踪信息

- [ ] **2.3** 建立候选人反馈日常流程 ⏰ 1月15-16日
  - 工作量：2小时
  - 内容：制定规范
    - 每周几更新候选人状态？
    - 谁负责发邮件跟进未回复的候选人？
    - 面试反馈如何及时记录？
  - 交付：《候选人管理日常规范》文档
  - 责任：符凌维

- [ ] **2.4** 用现有Skills做一次端到端的招聘演练 ⏰ 1月17日
  - 工作量：4小时
  - 内容：
    - 选择一个当前正在进行的招聘岗位
    - 用jd-generator重新生成JD（对比当前JD）
    - 用resume-screener重新筛选已有简历
    - 用interview-scheduler重新安排面试
    - 记录整个过程中的问题和时间节省
  - 交付：《Skills在招聘中的实际应用报告》
  - 责任：HR + Claude Code
  - 预期收益：
    - 发现现有Skills的不足
    - 为新Skill的开发提供需求
    - 看到实际的时间节省效果

|------|------|------|------|
